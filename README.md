# MyLibrary 书房

一个完全本地运行的个人小说藏书馆：Windows 双击即用，书架、阅读器、主题工坊、阅读花园，以及一只住在右下角的书库精灵 **Quill**。

> 所有数据（书库、阅读记录、记忆、贴图、白噪音）都存在你自己的电脑里，不上传任何服务器。

## 功能一览

### 书房 · 书架
- 按作者 / 题材分排陈列，置顶收藏作者，标签筛选，全库搜索
- 书脊配色系统：十套内置色卡 + 自建色卡（图片取色 / 手动编辑，无数量上限），一键整架重染

### 沉浸阅读器
- 进度记忆、书签、滚动/翻页、外观自定
- 支持 txt / epub / pdf；主题换肤、划线笔记、自动记进度这些阅读器专属功能面向 txt/epub，pdf 通过浏览器内置查看器展示
- **声音面板**：内置白噪音 + 自传音频，自定义分组、拖动排序、组合一键存取——给每本书配专属 BGM

### 主题工坊（保存才生效的草稿式编辑）
- 十一个位置独立换装：主背景 / 顶部插画条 / 底部横幅 / 左右侧饰 / 书桌 / 各页整页背景
- 定框图片编辑器：滚轮缩放、拖拽挪位、转角旋转，亮度/对比/饱和/色温/褪色/模糊/羽化全套
- 底色多停靠渐变 + 点晕染（圆 / 椭圆 / 环 / 星芒 / 方片 / 三角 / 六角 / 水滴 / 四瓣花 / 月牙 / 四芒 / 五角 / 心形，可调比例），纹理（宣纸 / 亚麻 / 斜纹 / 颗粒 / 帆布 / 细点），雾感磨砂，晕影
- 整套方案快照：背景 + 渐变 + 点 + 纹理 + 书脊色卡一键存取

### 阅读花园
- 每读一本种一棵树，随进度发芽开花；读完落成，重读旧书自动留档"往期树"
- 树种 / 树色自选，按色卡随机整块上色，草地底图逐月自定义

### Quill · 书库精灵
- 找书荐书、整理书架、读书笔记、章节总结、阅读计时（任何 OpenAI 兼容接口即可驱动）
- **玄学工坊**：八字排盘、大运流年流月流日流时、今日黄历宜忌、紫微基础盘、塔罗——排盘全部本地算法，AI 只负责解读。塔罗结合星座（西方占卜传统），每日一签结合生辰八字流日，两套体系分开，不混用
- **每日播报**：默认早上 8 点推送（可改时间），发不出去会每 15 分钟自动重试直到接口有额度；27 个预置新闻源勾选即订阅（BBC 中文 / NYT 中文 / FT / 少数派 / Nature / ESPN 等六类），也可自加 RSS
- **习惯打卡**：每天到点提醒 + 晚间小结；月历热力图、连续天数、目标进度条；连续达标授予奖牌
- **待办与日历**：聊天里说"提醒我周五交报告"即入待办，临期系统通知；待办和习惯打卡有统一日历（月/周视图，点哪天看哪天，当场编辑）
- **记忆**：自动记住生日、城市、喜好（可管理修改）；记下没聊完的心事，隔一天再聊时自然问一句进展
- **小游戏**：每日一签、猜拳、掷骰子、盲盒开卷（随机翻开一本没读过的书），胜负会记进 Quill 的记忆
- **陪伴细节**：读完一本书撒花、花园树特写；长按摸头会害羞；书架空闲时 Quill 会打盹；面板头顶一颗情绪光点，随互动和聊天内容变化六种状态和颜色；浮钮平时是羽毛，鼠标靠近换成当天的节气或节日插画（24 张专属图），报连续阅读天数；聊天消息支持撤回重编辑、对回复不满意可重新生成
- **多模态**：发图识图、表情包仓库（单张上传 + 批量导入）、维基配图
- 入口化身：羽毛 / 圆点 / 实时表盘 / 当日月相 / 萤火 / 叶子

## 快速开始

**Windows：** 解压后双击 `MyLibrary.exe`，浏览器/应用窗口自动打开 `http://127.0.0.1:8765`。

**源码运行：**
```bash
pip install fastapi uvicorn requests
python scripts/launcher.py
```

首次使用：设置页 → AI 配置，填入任意 OpenAI 兼容接口（DeepSeek / 中转站均可）→ 测试 → 保存。

## 目录结构

```
mylib/               后端（FastAPI 路由 + Quill 智能体 + 排盘引擎）
frontend/            前端（原生 JS，无构建步）
  home/js/           书房逻辑，按职责拆分为多个模块，后端按文件名顺序拼接下发
  reader/            阅读器（含白噪音 WebAudio 引擎）
static/themes/       内置纹理与主题素材
scripts/             启动器
tests/               pytest（59 项）+ 无头冒烟（36 项，打包前必跑）
library.db           你的书库与全部本地数据（升级时请保留）
```

## 工程结构

前端按职责拆分成独立模块，放在 `frontend/home/js/` 下：

```
10_shelf.js           书架 · 数据/搜索/详情/批量/下载篮/书单导出
20_quill.js           Quill 精灵 · 面板/对话/记忆/播报/待办/习惯/玄学
30_theme.js           主题工坊 · 背景方案/色卡/槽位/定框编辑器
40_journey_misc.js    时段主题/PWA/阅读历程/花园草地
50_share.js           收藏翻阅 / 分享书单
60_quill_extras.js    陪伴细节 · 佩饰/摸头/打盹/盲盒/庆祝/节气卡
62_term_art.js        24 张节气专属图
65_tarot.js           塔罗抽牌 · 牌扇/牌阵/抽牌手感
70_mood.js            情绪光点 · goo 流体六色六态
72_calendar.js        待办/习惯统一日历 · 月视图/周视图
```

页面请求 `/static/home.js` 时，后端按文件名顺序把这些模块拼接成一份下发，不依赖 webpack/vite 等构建工具，解压即用、改完刷新即生效。

`tests/` 下 59 项 pytest 覆盖后端接口（书库/主题/Quill 记忆/待办/习惯/节气/贴图导入/日历聚合/心事记忆/并发扫描/数据库迁移/星座换算/消息撤回重生成等），36 项无头冒烟测试在 Node 中执行完整前端脚本，回放主题/习惯/节气卡/塔罗/情绪光点/日历等场景；CI（GitHub Actions）每次推送自动运行，红灯拦截。

## 多设备部署（手机 / 平板当阅读器）

MyLibrary 本质是"一台主机跑服务，全家设备用浏览器读"。exe 只是 Windows 的打包壳，任何能跑 Python 的设备都能当主机：

**方案一 · 家里电脑当主机（零改动）**
Windows 开着 MyLibrary，手机连同一 Wi-Fi，浏览器访问启动窗口里显示的局域网地址（形如 `http://192.168.x.x:8765`），Safari/Chrome"添加到主屏幕"即得全屏应用。缺点是电脑得开着。

**方案二 · 安卓手机 / 平板当主机（Termux 一键）**
装好 [Termux](https://f-droid.org/packages/com.termux/)，把项目文件夹拷进手机，在 Termux 里 cd 到目录后一条命令：
```bash
bash 安卓一键部署.sh
```
首次运行自动装 Python、装依赖、在 `~/library` 建好空书库并拿唤醒锁保活；以后每次运行就是直接启动。本机浏览器访问 `http://127.0.0.1:8765`，其他设备访问手机 IP。注意到系统设置里给 Termux 关掉电池优化，否则息屏一会儿服务就没了。

**方案三 · 树莓派 / NAS / 旧电脑（Docker，最省心）**
```bash
docker build -t mylibrary .
docker run -d --restart=always -p 8765:8765 -v /你的书库:/data mylibrary
```
24 小时在线，全家设备随时可读，推荐长期方案。

> **安全提醒**：服务没有登录鉴权，谁能访问这个地址谁就能看你的书库和 AI 配置。只在家庭内网使用；出门在外想读，用 Tailscale / WireGuard 组虚拟局域网连回家，**不要**把端口直接暴露到公网。

## 升级与数据

覆盖新版本前先在设置页点「停止服务」；`library.db`、`_ambient/`（音频）、`_quill_stickers/`（贴图）都在项目根目录，覆盖时不要删除即可无损升级。
