# 更新说明 v3.53.0

接着把整理流水线没做完的往下做。这轮完成 **⑥ 版本更新沿用旧书的笔记/标签**。

## 去重时不再弄丢你在旧版本上写的东西 ✅
以前去重把重复的旧版本移走时，**直接删了那条记录**——如果你正好在被删的那本上写过简评、打过分、加过标签、划过线，就跟着没了。
现在改成：**删旧版本前，先把你的数据搬到"保留"的那本**，包括——
简评、评分、收藏、已读、最近阅读时间、标签、划线、书签、笔记、阅读进度。

搬运规则（稳妥、不乱覆盖）：
- **简评**：保留版本自己没写过，才用旧版本的（你写过的不会被盖）。
- **评分**：取较高的那个。
- **收藏 / 已读**：任一为是即为是。
- **阅读状态**：已读 > 在读 > 未读，取更靠后的。
- **最近阅读时间**：取更晚的。
- **标签**：两本的标签合并（并集、不重复）。
- **划线 / 书签 / 笔记 / 进度**：这些跟位置有关，跨版本可能错位，所以只在"保留版本一条都没有"时才整体搬过去，最稳妥。

逻辑已用造的数据单测：旧版本的简评/评分/收藏/状态/标签/划线/进度都正确搬到保留版本；保留版本若已有简评则不被覆盖、评分取高、状态取靠后。

去重卡片的说明也加了一句"数据会保留"，让你执行前心里有数。
