# 更新说明 v3.54.1（按你这次反馈修的）

## 改了
1. **「安装到桌面」不再多余**：只在真能安装时显示——手机浏览器（加到主屏）或桌面浏览器弹出安装提示时。你现在双击打开的这个独立窗口本身，不再显示这个按钮。
2. **进管理不再每次重算统计**：母库统计加了 60 秒服务端缓存 + 本地秒显上次结果。独立窗口里反复点「管理」会立刻出数字，不再每次全库重数十几万本（任务跑完会自动强制刷新）。
3. **管理页从上到下真的重排了**：按整理流水线顺序分组排：① 入库 → ② 完结+分章节 → ③–⑥ 作者整理 → ⑦ 去重 → ⑧ 打标签 →（底部）辅助/维护/找回。顶部导航点一步还是会跳过去。
4. **阅读不再白屏没反应**：打开书先亮出"正在加载正文… X%"，加载久了（30 秒）给提示；正文空/坏会明确报错。这样"半天加载不出来"至少能看到卡在哪。
5. **找回简评更全**：之前按 书名+作者 匹配，如果某本的作者被「合并笔名」改过，就匹配不上漏掉了。现在加了"按书名兜底"（书名唯一时也能认），更可能把你那条找回来。

## 关于你提到的几点
- **这本「理清章节」是 0 章**：它其实跑成功了，只是这本是短篇/一发完，正文里没有「第 X 章」这种标记（只认出 1 个番外），所以正文章节数是 0；「0 万字」是说不到 1 万字。不是坏了，是这本就没有分章。如果你这本明明很长却显示 0，那是文件读取的问题，请把书名告诉我单独查。
- **丢失的总评**：先去管理页「找回丢失的简评（从备份）」点「执行恢复」（这版加了按书名兜底）。如果那本作者被改过名，之前会漏，现在更可能找回。还是空的话，多半是当时没存进任何一份备份——那就只能从你手头别的备份库翻了。
- **算法优化**：这个我确实还没动——因为不知道具体是哪一步结果不对。请告诉我是哪一类不准：去重误删/漏删、作者认错、章节数不对、还是完结判断错？最好给一两个具体例子（书名+期望结果），我针对那一步调，而不是瞎改。
