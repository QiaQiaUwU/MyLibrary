# 更新说明 v3.61.0（把你早期那个能用的卷曲翻页找回来了）

谢谢你给早期版本——一看就明白了，是我之前几处改动把它弄坏的。早期能正常翻，靠两点：
1. 从 CDN 加载 StPageFlip 卷曲翻页库（你设备有网，手机截图里能看到 4G，所以那时一直能加载）；
2. 翻页调的是 `flipNext() / flipPrev()`（库自带的、带卷曲动画的方法）。

我后来的问题：①把手机排除在卷曲翻页外（强制用了简化翻页）；②把翻页改成了 `flip(target)`＋`turnToPage`——后者其实是"瞬间跳页、没动画"；③再后来干脆把 CDN 删了。三刀下去，卷曲就没了。

## 这版改回早期同款（并更稳）
- **恢复 CDN 加载** StPageFlip，并且**额外保留本地兜底**：先试本地 `frontend/vendor/page-flip.browser.js`，再试 CDN——联网用 CDN、纯离线放本地文件即可，两者都没有才退回内置 CSS 3D 翻页（也有立体翻页，绝不再"瞬间跳页"）。
- **翻页改回 `flipNext()/flipPrev()`**：真·卷曲动画回来了。
- **手机也用真卷曲**：去掉了之前把窄屏排除的逻辑，跟你早期一致。
- **页数上限 240 → 1500**：之前 240 会把《旧故新长》这种 235~259 页的书挡在卷曲翻页外，现在普通小说都走卷曲，只有上千页的超大整本才退回 CSS 3D 防卡。
- **分页同时适配卷曲盒子**：卷曲翻页的纸张盒子比普通页矮一点，分页时按更矮的那个来排，保证卷曲翻页里也不切最后一行。
