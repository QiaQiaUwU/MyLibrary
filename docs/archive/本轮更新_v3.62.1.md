# 更新说明 v3.62.1（找回简评 = 没有，我查了到底为什么）

## 先说结论：你的简评没有被程序弄丢
我把所有会写"总评(user_notes)"的地方都查了一遍：
- 在书里写总评（/api/update）：写法正确、会落库、会 commit ——**保存是好的**；
- 去重合并时：只在"保留的那本是空的"才把旧总评搬过去，不会拿空的覆盖你写过的；
- 导入 JSON / .mrpro：**只回填非空字段**，不会用空值把你已有的总评抹掉。
所以现在写的总评不会丢。你看到的那 2 条（旧故新长、关于主角团…）也是好好的。

## "找回"为什么是 0
「找回丢失的简评」只会找出"**备份里写过、但现在库里这本是空的**"那种。它扫不到东西，通常是因为：
- **以前根本没存过自动备份**——之前只有"去重/合并作者前"才会存一次备份。你要找回的那条要是不是在某次去重前写的，备份里就没有；
- 或者那本其实**没写过总评**。比如截图里《仙山深处有魔修》是打了 3 星**评分**、但**总评是空的**（评分 ≠ 总评，它可能本来就没写过总评）。

## 这版做的真正的修：自动备份
加了 **每天一份 library.db 自动备份**（开服务时后台存，用 sqlite 备份 API、兼容 WAL，保留最近 5 份，命名 `library.db.bak_auto_*`）。从今往后万一再丢，就能在「管理 → 找回丢失的简评」里从这些备份找回了。

## 你能帮我确认的
如果你**确定**给某本写过总评、现在没了——把**书名**发我，我直接帮你在现有数据库/备份里查那一条到底在不在、在哪。光靠"找回扫描"只能找回备份里有的，没备份的找不回，这是实话。
