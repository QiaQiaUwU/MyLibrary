# 更新说明 v3.62.6（这次真的把"两页/往前翻不了"修了）

你截图里仍是**左右两页并排**——这就是排版怪、往前翻不了的根。我前几次没修对，跟你说清楚这次到底改了什么、为什么这次对：

## 真正的根因
翻页用的是 StPageFlip。它判断"单页还是摊成两页"的规则大致是：**容器宽 ≥ 单页宽×2 就摊两页**。
而我们的**书容器一直是整个窗口那么宽**（满宽），单页才 680——宽屏上 1900 ≥ 680×2，它当然摊成两页。
我上一版只把单页上限调小，没动容器宽度，所以照旧两页。**这是我的疏忽。**

## 这次的修
1. **把书容器锁成"单页宽、居中"**（不再满窗），并把单页最小宽设成容器的 0.72——这样容器里**塞不下两页**，StPageFlip 只能单页竖版。1366 / 1920 / 2560 各种宽度都验算过，全是单页。
2. **点击翻页改成"按书分区"**：书居中、两边留白，于是——点**书的左 1/3（含左边空白）= 上一页**，右 1/3 = 下一页，中间 = 呼出菜单。以前翻页区是按整窗算的、落在远处空白里，所以你点书左边没反应、像"往前翻不了"。现在点书左半就能往前翻。键盘 ← → 也照常。

## 你重点试这两点
- 打开任一本书 → 应该是**居中的一页**（不再左右两页）；
- 点**书的左半边**能不能往前翻、右半边往后翻。

要还是两页：八成是没开到装了这版的文件夹（覆盖进 `D:\MyLibrary_v3`、保留 config.json），或阅读器窗口没重开。
