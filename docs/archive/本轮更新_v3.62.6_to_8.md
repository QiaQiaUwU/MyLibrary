# 更新说明 v3.62.8（终于把"改了没效果"的根挖出来了）

## 为什么我前几版"修了"你却还是两页、还是空白
关键发现：**你看到的阅读器一直在跑旧的 reader.js。** 桌面端的 webview / Service Worker 把旧的 reader.js 缓存住了，我改的代码根本没加载进去——所以两页、空白、往前翻不了，全是旧代码在跑。这不怪你刷新，是缓存太顽固。

这版从两头治：

### 1. 彻底破缓存（这条最关键）
- 阅读器页面和 reader.js 改成**绝不缓存**，并且每次启动都给 reader.js 加一个**新的版本号**（`reader.js?v=启动时间`）——URL 一变，缓存就必须重新取。书房主页 / home.js 同样处理。
- Service Worker 缓存版本号也升级了，旧缓存会在启动时自动清掉。
- **效果**：以后我改阅读器/书房的代码，你覆盖文件、重开，立刻就是新的，不会再"改了没反应"。

### 2. 翻页彻底改用"单页"，不再用那个老摊成两页的库
两页/空白/翻不回去，根子是之前那个翻页库（StPageFlip）在宽屏上自作主张摊成左右两页、还时不时空白。**这版直接弃用它**，换成我自己控的**单页立体翻页**：
- **永远单列、居中**（跟你截图里 Moon+ Reader 那种一样的单页），不再左右两页；
- 点**页面左 1/3 = 上一页**、右 1/3 = 下一页、中间 = 呼出菜单；键盘 ← →也行；
- 翻页/滚动来回切也不再空白。

### 3. 改状态后即时刷新（不用手动刷）
改阅读状态后任何视图都立即重排。（拖到作者/收藏/打标签/删除本来就实时；如果你说的"拖完要刷新"也是因为旧 home.js 缓存，这版破缓存后应该就好了——还不行告诉我具体拖什么。）
