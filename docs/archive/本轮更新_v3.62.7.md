# 更新说明 v3.62.7

## 1. 翻页/滚动切换几次后整页变空白 —— 修了（这是刚才那张黑屏图的病根）
切来切去会空白，有两个原因，都修了：
- **延迟初始化没设防**：切到翻页会等一两帧再建翻页书，可你要是这空档又切回滚动，那个回调还在隐藏的容器上建实例 → 空白。现在建之前先确认"此刻仍是翻页模式"，否则不建。
- **上一个翻页实例没清干净**：StPageFlip 销毁后会在容器上留类名/内联样式残骸，下次重建就空。现在销毁时把容器的类名和内联样式一并清掉，并在容器还没量到宽度时等一帧重试。
切了十几次也不该再空白了。要还有，告诉我你点的顺序。

## 2. 往前翻页：我查过了，是接好的
`flipPage(-1) → StPageFlip.flipPrev()`，键盘 ← 和"点书左1/3"都走这条。之前你觉得没反应，多半是**整页空白时点什么都没用**（就是上面那个 bug）。这版空白修好后，请这样验证往前翻：
- 点**书页左边三分之一**（或按键盘 ←）→ 应回上一页；
- 右三分之一（或 →）→ 下一页；中间 → 呼出菜单。
要还是翻不回去，告诉我"点了书左边/按了←、有没有任何反应"，我再深挖 flipPrev。

## 3. 移动/改状态后不用再手动刷新
改阅读状态（未读→在读→已读）以前只在"书架"视图才实时刷新，别的视图要手动刷。现在改完**任何视图都立即重排**，书直接挪到对应分区。
（拖到作者、拖到收藏、打标签、删除——本来就已经是实时的。）

## 4. 关于"排版自适应"
说明一下：分页**不是**靠你给尺寸写死的。它每次都**实时量当前窗口/页面盒子的真实宽高**来排版（你拉窗口大小会自动重排），单页宽也是按窗口算的。第 1 点的空白、上一版的两页，都是具体 bug，不是"没做自适应"。我上次说"要尺寸"只是想兜底排查极端情况，不是要写死——你不给我也照样自适应。
