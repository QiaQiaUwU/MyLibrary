# 更新说明 v3.64.6

## 1. 页面底部空一大块——修好了
原因：分页时为了迁就卷曲翻页库的盒子（之前被设成比屏幕矮 100px），把每页都压矮了约 140px，所以底部空一截。现在让卷曲翻页的盒子和普通页面**一样满高**，分页按满高来排，**底部不再空**。

## 2. 翻页没动画——很可能是这个原因
内置翻页用了"双 requestAnimationFrame"来启动动画，这在某些 webview（你桌面端就是）里会被直接跳到结尾 = 看着没动画。这版改成**强制回流**再启动过渡（最稳的写法）。翻页时那一页会绕书脊翻折，应该能看见了。

> 还有一种可能：**Windows 的"显示动画"被关了**。请到 设置 → 辅助功能 → 视觉效果，看看「动画效果」是不是开着——关了的话，整个系统里所有 App（包括这个）的动画都会被系统掐掉，那不是程序的问题。麻烦你确认一下。

## 3. 想请你确认的两件
- **移动端翻页**：你手机上是「翻页」模式还是「滚动」模式？（阅读器里点一下呼出菜单 → 外观里能切）翻页模式下，点屏幕右 1/3 翻不翻？
- **拖动后不刷新**：到底是把书拖到哪里？(拖到作者那排 / 拖到收藏 / 拖到题材标签 / 还是详情里点状态)——这几个我都加了即时刷新+存缓存，但你那个一直不行，说明是某个我没覆盖到的路径。你说一个具体的，我立刻钉死。
