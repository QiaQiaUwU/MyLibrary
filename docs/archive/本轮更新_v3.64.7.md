# 更新说明 v3.64.7 — 把"柔软翻页 + 翻书声"按你早期能用的样子修回来

我翻了之前的聊天记录，确认了两件事，也找到了被我改坏的地方：

## 1. 翻书声——之前一直没响，是因为音频文件根本没在包里
翻页音效是去 `/api/sfx/pageturn` 取文件，但那个文件（`_sfx/pageturn.mp3`）从来没随包发布，要你自己放——所以你这边一直是静音的。
这版我**合成了一段翻书声**（纸张沙沙+颤动的那种，约 0.5 秒）直接打进包里：`static/_sfx/pageturn.wav`。**开箱即响，不用你再找音频**。翻页时（卷曲翻页和内置翻页都算）就会播。
> 你要是有更好听的，把自己的 `pageturn.mp3` 放到母库的 `_sfx/` 下，会优先用你的。

## 2. 柔软卷曲翻页——是被我后加的"安全网"误杀了
那个真·卷曲效果是 StPageFlip 库做的，你早期能用。后来我加了一条"**一检测到两页就立刻销毁退回**"的保险，这条太敏感——在页面布局还没稳的一瞬间会误判，把本来正常的卷曲翻页**当场杀掉**，退回成生硬的内置翻页。这就是你说的"短暂修好又被改坏"。
这版**去掉了那条会误杀的保险**（单页竖排本来就用 `usePortrait` 强制了，不需要它），卷曲翻页一旦库加载好（联网走 jsdelivr，你那边能连）就会自己上来，跟你早期一样的软翻页。

升级：覆盖 `D:\MyLibrary_v3`（留 config.json）→ 结束 python/pythonw → 启动 → **阅读器窗口关掉重开** → 翻一页听听有没有翻书声、看翻页是不是软卷曲。
（翻书声音量/开关在阅读器外观设置里；要是没声音先确认那里没被关。）
