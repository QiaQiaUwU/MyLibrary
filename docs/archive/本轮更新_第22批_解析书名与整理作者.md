# 本轮更新（第 22 批）— 解析书名 / 整理作者名（根因修复）+ 管理排版

你说得对：**作者分类乱、找回作者、查重不准，根子都在"书名/作者名没解析好"**。这轮从根上做了处理。

## 1. 新增「整理作者名 / 解析书名」任务 ✅（管理页）
把入库时被错当成作者的"描述残留"清成核心作者名，并把变体合并：
- 「淮上-补车番 / 淮上肉章 / 淮上近代架空 / 破云by淮上」→ 都归到 **淮上**
- 「priest完结番外三篇 / priest完结+番外 / priest2019.6.16更新番外」→ 都归到 **priest**
- 「火风L3p / 火风L补全番外强推 / 火风L换攻 / 火风L番外全EO文…那就便宜别人了※※」→ 都归到 **火风L**

做法（已用你截图里的真实例子单测通过）：
- 识别并去掉描述词（完结/番外/补车番/肉章/近代架空/全本/换攻/更新/强推/全EO文…）、日期（2019.6.16）、括注、※及其后的整句；
- `byXX` / `XX著` 取作者部分；
- 再把"核心作者 + 纯英文数字尾巴"（如 火风L3p、火风L）并到同一个。
- 正常作者名（送泥一条鱼、priest、诗无茶）**保持不动**。
- **先预览**（看会规范多少本、合并多少个杂名、给 40 条示例），确认后**执行**会先自动备份数据库再写回。

整理完，**作者分类会立刻清爽**，之后的**查重、找回作者也会更准**（因为作者对上了）。

> 用法：管理页 →「整理作者名 / 解析书名」→ 先「预览可整理的」→ 确认后「执行整理」→ 回书房强刷。

## 2. 找回作者：上一批已改严（顺带说明）
找回作者现在只在作者名出现在书名结尾/紧挨分隔符时才认，减少认错。配合这次的「整理作者名」一起跑效果最好：**先整理作者名 → 再找回佚名 → 再查重**。

## 3. 管理页排版 ✅
- 把一长串卡片**分组**加了小标题：**入库·收集散落 / 查重·去重·版本 / 分类标签·作者整理**，不再是一坨平铺。
- 顶部原本就有 总书目 / 作者 / 已打标签 / 未打标签 / 带走清单 的统计；各任务的"预览"也会列出具体情况（能找回多少、能整理多少、有多少疑似异常重复组等）。
