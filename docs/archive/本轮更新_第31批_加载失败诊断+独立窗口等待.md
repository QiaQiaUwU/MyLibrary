# 本轮更新（第 31 批）

## 1. 「加载失败，请检查服务」——现在会告诉你真正原因
你的截图说明：**页面（书房界面）能打开 = 服务在跑**，但**书的数据没取到**（/api/shelf 出错）。
代码本身没问题（各种情况都测过不崩），所以是运行时取数据失败——**最常见的是放书的那块硬盘（F盘）没插好或盘符变了**，数据库打不开。

以前这种情况只显示一句「加载失败」，看不出原因。现在会把真正原因显示出来，比如
「打不开藏书数据库（… unable to open database file）。请确认放书的硬盘已连接、盘符没变…」。

### 你可以马上自查（不用等更新）
在浏览器地址栏直接打开：`http://127.0.0.1:8765/api/shelf`
- 如果显示一大串带书名的 JSON → 数据正常；
- 如果显示报错 / "Internal Server Error" → 多半是 F 盘没连上。把那段文字发我。

并确认：放书的硬盘（之前是 F:\MyLibrary）**已插好、盘符还是 F**。若盘符变了，去「设置 → 藏书目录」改成新路径。

## 2. 独立窗口：等服务起来再开，避免一开就「加载失败」
之前窗口是固定延时 1 秒就打开，13万本的库初始化要更久，于是窗口先开、服务还没好 → 加载失败。
现在改成**轮询等服务真正能响应了再打开窗口**，就不会出现"刚开就失败"。

> 升级到 v34：先在任务管理器结束所有 python.exe/pythonw.exe（旧版残留），再启动；之后按 Ctrl+Shift+R 刷新两次。
