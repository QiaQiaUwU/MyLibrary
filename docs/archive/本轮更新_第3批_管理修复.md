# 本轮更新（第 3 批）— 管理/去重/作者/标签

针对你截图里管理页那几个硬伤，逐个改了。**重要前提**：我这边没有你的 13 万本库、也跑不起真实服务，所以下面是“读代码定位 + 改逻辑 + 用造的小样本单测”得出的结论；**请覆盖后在真库上重跑这几个任务验证**，把结果告诉我。

## 1. 去重“保留”留错版本（截图最明显的 bug）
- 原因：选哪本保留时只看 `(收藏, quality_score, 字数)`，而 `quality_score` 不准，于是把**章节更少**的留下了（你截图里 140 章/81.5 万字被保留，162 章/88.7 万字反而被标清理）。
- 已改：去重时**先数每本的章节/番外**，再按 **章节 > 番外 > 字数 > 文件大小** 选最全的那本保留（收藏的仍优先保住）。预览里显示的“保留”就是真正最全的版本。

## 2. 完结识别“点继续扫描永远只给一条、说全扫过了”
- 原因：用 `chapter_count` 是否为空来判断“扫没扫过”，但导入时就把它填上了，于是续跑永远认为“全扫过”，其实没真扫。
- 已改：改用**专门的 `proc_finish_at` 标记**（新加的列，初始全空）。所以“继续扫描”这次会**真的从头逐本扫**、有真实进度；扫一本标一本，中断能续。“重新全扫”会清标记重来。
- 还加了**扫描记录**：每次扫描在 `processing_batch` 表写一行（总数/已完成/扫到第几本/状态/时间），就是你要的“每次扫描的记录 + 进度备份”。

## 3. 作者重名分类碎得离谱（诗无茶被拆成 7 个）
- 原因：自动合并只会剥 `_数字` 和 `-补番`，像「诗无茶古代」「诗无茶-番全」「诗无茶补番」「: 诗无茶」「诗无茶--」一律没合并。
- 已改：改成按**“核心名”归一**——去掉首尾标点/符号、去掉结尾装饰词（番外/番全/补番/古代/现代/完结/原版/出书…）和编号，核心名相同的**归到书最多的那个名**。
- 用你截图里的例子单测过：`诗无茶古代 / 诗无茶补番 / 诗无茶-番全 / 诗无茶-- / : 诗无茶 / 诗无茶完结 / 诗无茶_1` → **全部并到「诗无茶」**。
- ⚠️ 这一步只改了“预览/执行用的别名表”，**真正合并要你点「执行合并」**才会落库（之前会备份 `library.db.bak_before_merge`）。

## 4. 找回作者扫得太少（几乎没找回）
- 原因：只扫**清洗后的书名**，而作者常留在**原始文件名**里；命中的若是带后缀的碎片名也没清干净。
- 已改：找回时**书名 + 原始文件名都扫**；命中的作者名再清一遍后缀（`诗无茶_1`→`诗无茶`）。
- 单测：`婆婆_诗无茶_1.txt`、`沉火不眠诗无茶完结番_1.txt`、`旧故新长 作者诗无茶`、`娑婆by诗无茶` → **都能找回「诗无茶」**。

## 5. 标签筛选（你要挑虫族文看）
- 书架顶部加了**标签筛选条**：从全库标签里列出常见的，**可多选叠加筛**（小说标签是复数的，选多个就是“同时含这些标签”）。按作者/按题材的大类分组**照旧**，标签只是在上面做筛选。
- 想看虫族文：等 AI 打标签把“虫族”标上后，点一下“虫族”标签即可只看这些。
