## 三、整理流水线起步：按正文找回 / 合并作者（笔名归一）
你点名最难的一环——"同一作者不同笔名，靠比对正文判断"。做出来了，**预览优先、执行前自动备份数据库**。
- 原理：同一作者的书正文开头常带身份标记（**微博@、lofter、by XX、作者：XX、首发平台**）。扫每本书正文开头，把"**标记相同、但记录的作者名不一致**"的书聚成一类，归一到同一作者。
- 你的例子已通过单测：正文里有「\*微博：@送泥一条鱼」的两本书，作者分别记成「抗病毒口服液」和「送泥一条鱼」，会**认出是同一人并合并到「送泥一条鱼」**（归一名优先选与标记一致的那个笔名）。
- 入口：**管理页 → 「按正文找回 / 合并作者（笔名归一）」** 卡片。先点「预览可合并的」，会列出 `正文标记 / 这些笔名 / → 归为`；确认后点「执行合并」（先自动备份 `library.db.bak_时间戳`）。
- 新增：`mylib/tools/author_from_content.py`（extract_handles / find_clusters / apply_clusters，均单测通过）、`task_author_by_content` 任务、`/api/admin/task/author_by_content` 路由、管理页卡片。
- 提醒：要逐本读正文开头，**13 万本会跑几分钟**，预览跑完不写库，放心试。
