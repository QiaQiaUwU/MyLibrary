## 2. 编辑模式现在做齐了
点右上「选择」进编辑模式，底部工具条新增三个按钮：
- **打标签**：给选中的书统一打一个题材标签（会进题材筛选）。
- **置顶作者**：把选中书的作者一次性加进「收藏作者」并置顶。
- **删除**：批量删除，文件移到 `书库/_recycle/`（不直接抹，可找回），二次确认。

后端新增 `POST /api/books/batch`（动作：tag / untag / delete / favorite），批量操作走一个请求，效率和安全都更好。

## 3. 拖动归类（你要的“拖到题材上”）
进编辑模式后，**顶部出现一排「题材放置目标」**（库里最常见的题材 + 「＋新标签」）。
- 选中几本书，**拖到某个题材上**——那个目标会**高亮、放大、描边**，松手就给这些书批量打上该题材标签。
- 想要新标签直接点「＋新标签」输入即可。

## 4. 管理页：处理进度
管理页顶部新增 **「处理进度」卡**，按任务显示**已完成 N / 共 M（百分比）+ 上次跑的时间**：
- 去重 / AI打标签 / 元数据·作者 / 完结判定，各一条进度条。
- 数据来自每本书上的 `proc_*_at` 标记列（跑过哪本就记一笔），上次时间来自 `processing_batch`。
- 这样你一眼能看出“整库还剩多少没处理”，不用瞎跑。

> 进度卡要显示数字，前提是数据库已迁移出 `proc_*_at` 列（启动会自动迁移）；刚换库或没跑过任务时显示 0 / 还没跑过，正常。
