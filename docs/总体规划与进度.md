# 项目结构与版本计划（唯一总账）

> 这份是唯一的"看这里就够了"文档。历史逐批更新说明已归档到 `docs/archive/`。

## 版本（格式定死，不再换）
用业界通用的 **语义化版本 v主.次.修**（SemVer），就这一套，以后不变：
- **主**（现在=3）：大改、推倒重来才动。
- **次**：加了新功能就 +1（次 +1 时，修归 0）。
- **修**：只是小修小补 / 调样式 / 修 bug，就 +1。
- 例：v3.50.0 → 加功能 → v3.51.0 → 只修个小问题 → v3.51.1。
- **当前：v3.51.0**。代码包叫 `MyLibrary_v3.51.0.zip`；每版在 `docs/更新日志.md` 顶部记一段。

```
mylib/
  core/      mylib_core(建库/schema)、migrate_db(迁移，幂等)
  server/    _state(共享:连接/正文缓存)、frontend_loader(实时读盘)
    routes/  books / reader / journey / admin / collect / settings / quill / theme / media / pages
  agents/    mylib_agent(阅读助手) / quill_agent(书库精灵+会话)
  tools/     dedup_library 去重 / merge_authors 笔名合并 / recover_authors 找回作者(书名)
             / author_from_content 找回作者(正文)[新] / chapter_counter 章节计数
             / classify_genres 标签词库 / build_search_index / watch_inbox 自动收拢
  utils/     mrpro / booklist_export 等

  home/      书房+书架+阅读历程+管理入口（home.js 主逻辑）

docs/        总账(本文) + 更新日志 + GitHub指南 + archive/(历史逐批)

```

## 数据库要点
- books：去重三件套(content_hash/raw_hash/quality_score)+ user_notes/rating + reading_status/is_finished/chapter_count/extra_count + last_open + proc_* 处理标记。
- 关联表：reading_progress / reading_diary / highlights / bookmarks / reading_notes / tags / book_tags / duplicates / processing_batch / quill_sessions / quill_messages / books_fts(全文)。

## 路线图
### 已完成（近几轮）
- 简评/总评找回 + 阅读记录置顶「我的总评」+ 书架最新在读放前
- 阅读器划线重做（字符偏移定位/就地重绘/长段可划/换色）
- 翻页前后都有动画 + 窄屏适配修「下面半页空」
- 搜索与筛选分离 + 漏斗独立一行可折叠 + 多选栏深色 + 退出常驻
- 移动端"加载不全"修复 + 「安装到主屏/桌面」入口默认可见
- 手动「理清章节」按钮（解析并存库）
- Quill 历史对话（持久化+列表+回看+删除）
- 拖动作者归类自动改作者名
- 阅读进度条可直接点/拖跳读
- 阅读记录的「种树」也按最新阅读排前（与书架一致）
- 「生成书单」移到阅读记录界面（不再在多选栏）
- 每本书的「阅读回顾」：哪些天读过、估算读了几遍（第二/三遍的时间）

### 整理流水线（按此顺序推进）
1. [done] 找回作者(书名) recover_authors
2. [done] 找回作者/合并笔名(按正文) author_from_content —— 扫正文里的作者标记（微博@/lofter/by/作者：/首发平台）聚类，把同一作者不同笔名归一。预览优先、可撤销。
3. [done] 分章节 —— 已并入「完结识别 + 分章节」批量任务（task_detect_finished：全库扫描存 chapter_count/extra_count + 完结状态），另有详情里手动「理清章节」按单本补。
4. [done] 去重 —— 三层去重(哈希/同作者书名相似/空壳)已有；本轮补上"删旧版本前把简评/评分/标签/划线等搬到保留版本"。
5. [done] 去重后再打标签（省 token）—— 打标签默认范围就是"未打标签的书"，去重后只给没标过的标，不重复花钱。
6. [done] 版本更新沿用旧书标签/信息/笔记 —— 去重(也即版本更替)时自动迁移用户数据，见 _carry_user_data。
7. [done] 管理面板「由浅入深」重排 —— 顶部加了「整理流水线·推荐顺序」导航(1→7)，点一步跳到对应卡片并高亮；卡片已加锚点。后续可再做成真正分步向导。
