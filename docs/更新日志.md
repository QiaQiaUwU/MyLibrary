# 更新日志（新→旧）

## v3.64.21（翻页彻底重写为最稳做法 + 修打包 Qt 冲突 + 换图标说明）
- **翻页"闪一下不动"——根因 + 重写**：上一版(.20)用"`.page-sec` 分页 + `scrollIntoView`"翻页，分页是按另一个盒子(`.page-sheet` 680px)算的、塞进滚动容器后高度对不上，`scrollIntoView` 经常定位不动，但淡入动画照放 → 就是"闪一下没反应"。本版把这套整个删掉，改为最稳的做法：**翻页与滚动共用同一套连续正文(.rblock)**（划线/定位完全一致）；翻页模式用 CSS `overflow:hidden` **禁掉手动滚动**（满足"翻页时不该能往下滚"），点左右用脚本 `scrollTop` 把容器**整屏换到上/下一页**（`overflow:hidden` 不挡脚本滚动 → 一定换得动），配 0.14s **交叉淡入淡出**遮住跳变 → 读着是"翻了一页"而非"在滚"。每翻一页 = 一屏正文、留一行重叠（不丢行/不切行）；连点有锁防错位。
- **翻页划线跨页续滚**：划线拖到屏幕下沿时，计时器照旧自动续滚（也走脚本 `scrollTop`，`overflow:hidden` 下有效）→ 可跨页连续选到需要位置；松手即停。
- **翻页↔滚动切换零跳变**：两种模式 DOM 完全一样，切换只切"能否手动滚 + 点击是否换页"这个行为，不重渲染、不跳位、正文停在原处。
- **打包报错"multiple Qt bindings"——修**：你这台全局 Python 同时装了 PyQt5 和 PyQt6（科研环境的其它包），PyInstaller 不允许一个程序打两套 Qt → 直接中止。`MyLibrary.spec` 加 `excludes` 排掉本应用根本不用的 Qt/numpy/matplotlib/pygame/IPython/PIL 等大包（已逐个核对无 import）→ 修掉中止，exe 还更小。
- **`build_exe.bat`**：全程写 `build_log.txt`；并修掉"既报 DONE 又报 NOT produced"的括号转义 bug（改用 goto 标签，成功/失败只报一句）。
- **换图标**：exe 图标取自 `static/app.ico`。换图标 = 用自己的 `.ico` 覆盖这个文件后重打即可（没有 .ico 工具的话，给一张 PNG/JPG，可代为转成多尺寸 .ico 放进去）。

## v3.64.20（在 v3.64.19 上：翻页改"瞬间换页" + 加云端打包）
- **翻页"不像翻页"——改对**：上一版翻页是"平滑滚动一整页"，看到的是滚动动作、读着仍像在滚。改为**瞬间翻到上/下一整页**（scrollIntoView 直接定位、无滚动过程，像 Kindle 啪一下换页）+ 一个 0.2s 极短淡入，读着就是"翻页"。点左右/方向键各翻一页；划线拖到页底续滚跨页不变；滚动模式不动。
- **云端打包（你零安装）**：新增 `.github/workflows/build-windows-exe.yml`。推到 GitHub 后，在仓库网页 Actions 里点一下 Run workflow，GitHub 的 Windows 机器会自动打包，几分钟后在 Artifacts 里下载 `MyLibrary.exe`——你本地不装 Python、不敲命令。

## v3.64.19（基于你手上的 v3.64.16，专修三处：划线定位 / 拖书无反应 / 翻页）
- **划线定位差很多——真修**：① 开书时统一换行符（\r\n/\r→\n），让"原文字符偏移"和屏幕渲染严格对齐；② 不再只校正"没有位置的老划线"，改为开书时按每条划线的文字在正文里重新定位（recoverHlOffsets 全量重锚），历史版本里按错误偏移存下的划线（带着错的 start）也会自动归位，不必删了重画；③ 选区取偏移补上 .page-sec 锚点。
- **拖书换作者"完全没反应"——根因（确定）**：搜索状态下界面用 SEARCH_RESULTS 渲染，而 dropToAuthor 只改了 DATA（两份是不同对象），所以拖确实生效、改的那份却没被画出来。新增 updateBook() 让改属性时 DATA 与 SEARCH_RESULTS 两份同时改（dropToAuthor／收藏／打标签均走它）→ 搜索状态下拖完立刻看到移动 + 目标排高亮闪。
- **翻页改回真·一页页翻**：删掉"点右边平滑滚一屏"。翻页模式改为复用按行测高分页 paginate() 把整本切成"整页"（.page-sec，每页=视口可读高、断点在行边界不切行），点左右／方向键＝翻一整页（吸附）。只有划线选区拖到屏幕下沿那一下才自动续滚到下一页、可接着划到合适位置（跨页划线）；松手仍是翻页。**滚动模式完全没动，是安全兜底。** 翻页↔滚动切换、改字号/行距/字体、缩放窗口都会重排并回到当前阅读位置。

## v3.64.11
- 终于定位并修好"小窗最大化后整页空白"(真·根因,在 page-flip 库):库的 setHandlers() 总会 window.addEventListener('resize',onResize),但 destroy() 只有在 useMouseEvents 为真时才调 removeHandlers() 去摘它;我们用 useMouseEvents:false → 那个 resize 监听【永不被移除】、一路累积。最大化时旧实例的 onResize 仍按"放大后的容器 vs 建实例时的固定页尺寸"重算把页面挪出可视区 → 整页空白、页码冻住不变。修法:① destroyFlip 在 destroy 前临时把 useMouseEvents 置真,逼库执行 removeHandlers() 摘掉它注册的 resize 监听(顺带要移除的鼠标监听本就没注册,无害);② 我们自己的 resize 监听(脚本初始化即注册,早于库的 onResize)里先 destroyFlip——本次 resize 里它先执行并摘掉库的监听,库的 onResize 还没轮到就被移除,于是最大化不再挪位空白,随后防抖按新尺寸重建。
