## v3.68.2（划线/笔记跳转定位——真修复）
- **跳转落不到准确位置（手机 / 电脑都是）——根因找到并修**：正文块用 `content-visibility:auto` 只渲染可视块，未渲染块用占位高度 `contain-intrinsic-size`（原为死值 600px）。但一个 4000 字的块真实高度常两三千 px，占位太矮 → 计算目标"绝对坐标"时严重偏短，且该坐标随滚动/哪些块被渲染而漂移，导致跳转落偏（模拟实测误差约 4000+px）。修法两处：① 跳转改用浏览器原生 `scrollIntoView` 把目标滚进视口（原生正确处理 content-visibility 占位 + 滚动锚定），再做至多两次小幅微调落到精确字符行；② 首屏渲染后实测首块真实高度，写入 `--blk-ih` 作为所有块的占位高度（占位≈真实），并保留 `auto` 关键字让块渲染过一次后记住各自真实高度。模拟验证：占位准确后 20/20 落点误差 ≤2px。
- **.gitignore**：补充排除个人迁移笔记（安装与迁移指南.md）与运行日志（`*_log.txt`），上传 GitHub 时不带这些。

## v3.64.24（目录高亮在读章节 + 去重逐页确认 + 作者找回增强 + 佚名跨状态查重）
- **目录高亮正在读的章节**：打开目录时，当前阅读位置所在的章节会高亮并标"在读"，还会自动滚到可视中间，长书里一眼就知道读到哪一章。
- **去重可逐页翻看、逐组确认**：以前预览只显示前 40 组、其余看不到。现在预览上限提到 3000 组并加了翻页，可一页页翻、一组组确认；任意一行点一下即可改"保留"，跨页都记得住。
- **作者找回增强（治本）**：以前只认书名里的显式标记（作者/by/@）或库里已有的作者名，"书名_作者"这种没标记、作者又还不在库里的大量佚名书都漏了。现在加了按书名词频推断——同一作者名作为首/尾段反复出现在多本【不同书名】里时，认作疑似作者补进匹配集（"多本不同书共用同一名"这条约束能挡掉番外/完结/合集这类反复出现但非作者的词）。仍走预览，过目后再应用。
- **佚名/空作者跨状态查重（补全）**：查重原本按"同作者"分组，佚名书和有作者的同一本书永远不在一组、查不出重复，这是漏的大头。现加第四层：同一书名下若恰好 1 个真实作者加若干佚名/空作者，就把佚名那些当作丢了作者的同一本合并；全佚名（无真实作者锚）的同名书容易撞名，稳妥不在这里合并；多个真实作者撞名也跳过。配合"不确定就并存"，撞名也不会误删。
- 建议顺序：先跑「完结识别」补章节 → 再跑「从书名找回作者」（现在能找回更多）→ 最后「查重」（作者补全后能查得更全）。

## v3.64.23（去重保留算法纠偏 + 不确定多留版本 + 完结识别显示全库累计）
- **去重老留错版本——纠偏**：保留哪一版以前把"字数"当第一信号，但字数常有过时脏数据（0.0MB 的残档却标着 1.7万字），于是把空档误选成保留、把真正更全的 9章 版清掉。现改为以"文件大小"为主信号（最实在的内容量），再看章节、字数；近乎空的文件直接垫底。截图里"哪个龟儿乱扔娃娃""我喜欢上了一个怂货"那种留错的，已纠正。
- **不确定的重复，多留几个版本**：以前每组只留 1 本、其余全清，但有时几个版本各有所长（一个章节多、一个字数多），机器拿不准谁更全。现改为只清两类——近乎空/碎片，以及在 大小+章节+字数 三项上都不优于保留版的（严格更差）；其余在某一维更优的会标"并存"一并保留，不再误删。预览里能看到"保留 / 并存 / 清理"三种，点任意一行可手动把它设为唯一保留。
- **完结识别看着只有 2 本——其实是显示问题**：完结/连载/未知的计数以前每次运行从 0 算、只统计"本次扫的书"。13 万本若之前已扫完，这次续跑只新扫了 2 本，就显示"完结 2"，看着像全库只有 2 本完结。现在改为同时统计并显示"全库累计：完结 X、连载/未完 Y、未标记 Z"，是真实总数。

## v3.64.22（exe 自动退出 + 手机划线修复 + 阅读进度真实统计）
- **关掉浏览器后进程不退、再开打不开——修**：exe 的 launcher 以前既不查端口占用、也没有退出机制，旧实例一直占着端口，再点 exe 就 bind 失败、像"打不开"，得去任务管理器杀。现加两道：① 启动时若端口已被占用（说明已经在跑），直接打开浏览器连过去、不再重复启动；② 心跳自动退出——网页每 20 秒 ping 一次，看门狗发现 90 秒没人 ping（浏览器全关了）就自己退出进程、释放端口。手机锁屏/切后台时定时器降频，90 秒宽限不会误杀；切回前台立刻补一次心跳。
- **苹果手机划线划不上、还自己往下滑、点不开菜单——修**：选字续滚那段是给桌面鼠标拖拽设计的，在 iOS 上会和 Safari 原生的长按选字 + 选字滚动打架，形成双重滚动，于是选不上、页面乱滑。现加触摸设备判定，手机/平板上一律交给系统原生选字，不再用自定义续滚。另外划完留下的选区以前会把后续点击也拦掉，导致点不开菜单；现改为干净轻点会先清掉残留选区再正常呼出菜单。
- **在读的书进度全是 30%——修**：书房接口 `/api/shelf` 以前根本没读 `reading_progress` 表，前端只能拿"阅读时长"瞎估、估不出就兜底成 30%。现接口按每本最新一条真实进度返回（优先用存的百分比，没有就用 阅读位置÷总字数 现算），在读无记录的显示 0% 而不是假的 30%。

## v3.64.21（翻页彻底重写为最稳做法 + 修打包 Qt 冲突 + 换图标说明）
- **翻页"闪一下不动"——根因 + 重写**：上一版(.20)用"`.page-sec` 分页 + `scrollIntoView`"翻页，分页是按另一个盒子(`.page-sheet` 680px)算的、塞进滚动容器后高度对不上，`scrollIntoView` 经常定位不动，但淡入动画照放 → 就是"闪一下没反应"。本版把这套整个删掉，改为最稳的做法：**翻页与滚动共用同一套连续正文(.rblock)**（划线/定位完全一致）；翻页模式用 CSS `overflow:hidden` **禁掉手动滚动**（满足"翻页时不该能往下滚"），点左右用脚本 `scrollTop` 把容器**整屏换到上/下一页**（`overflow:hidden` 不挡脚本滚动 → 一定换得动），配 0.14s **交叉淡入淡出**遮住跳变 → 读着是"翻了一页"而非"在滚"。每翻一页 = 一屏正文、留一行重叠（不丢行/不切行）；连点有锁防错位。
- **翻页划线跨页续滚**：划线拖到屏幕下沿时，计时器照旧自动续滚（也走脚本 `scrollTop`，`overflow:hidden` 下有效）→ 可跨页连续选到需要位置；松手即停。
- **翻页↔滚动切换零跳变**：两种模式 DOM 完全一样，切换只切"能否手动滚 + 点击是否换页"这个行为，不重渲染、不跳位、正文停在原处。
- **打包报错"multiple Qt bindings"——修**：你这台全局 Python 同时装了 PyQt5 和 PyQt6（科研环境的其它包），PyInstaller 不允许一个程序打两套 Qt → 直接中止。`MyLibrary.spec` 加 `excludes` 排掉本应用根本不用的 Qt/numpy/matplotlib/pygame/IPython/PIL 等大包（已逐个核对无 import）→ 修掉中止，exe 还更小。
- **`build_exe.bat`**：全程写 `build_log.txt`；并修掉"既报 DONE 又报 NOT produced"的括号转义 bug（改用 goto 标签，成功/失败只报一句）。
- **换图标**：exe 图标取自 `static/app.ico`。换图标 = 用自己的 `.ico` 覆盖这个文件后重打即可（没有 .ico 工具的话，给一张 PNG/JPG，可代为转成多尺寸 .ico 放进去）。
