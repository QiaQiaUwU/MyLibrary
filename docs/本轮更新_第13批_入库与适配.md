# 本轮更新（第 13 批）— 批量入库、可折叠筛选/任务条、手机端适配

逐条回应 Point 9–12：

## 9. 自动入库在哪 / 批量入库怎么做 ✅
- **批量入库**：打开「管理」→ 顶部新增的「**批量入库（扫描文件夹）**」卡片，填一个文件夹完整路径（含子文件夹、压缩包都行），点「开始入库」——会自动解压、去重、入库，进度在卡片里走。它用的是给你建库时同一套扫描逻辑。
- **少量书**：直接把文件**拖到「书房」页面**就能导入。
- **自动入库（放进文件夹就自动收）**：命令行跑
  `python mylib/tools/watch_inbox.py 你的库路径 --all`（先 `pip install watchdog`）。
  它能盯着投递口 + 微信/QQ/浏览器下载等目录，识别小说自动入库，跳过聊天记录/文档/图片。
> 提醒：批量入库会把文件去重后移进书库。第一次先拿一个小文件夹试，确认没问题再上大批量。

## 10. 标签筛选可折叠 ✅
筛选面板改成**按维度折叠**：默认全部收起，只显示维度名 + 数量；点维度名展开/收起，右上「展开/收起全部」一键切换。这样就算所有标签都跑出来了，面板也不会占满整屏。

## 11. 管理在跑的处理可折叠到边上 ✅
书房左下那条后台任务进度，加了「－」**折叠成一个小条**（会记住折叠状态，下次还折叠），点小条即可展开。这个进度是**服务端轮询**的，所以你在手机/别的设备打开书房，只要任务在跑就都能看到，行为一致。

## 12. 手机端适配 ⚠️（部分修好，部分需真机继续调）
已修：
- **字整体太靠上** → 顶部留出了状态栏/刘海的高度（safe-area），正文不再顶到最上面。
- **中间一块白** → 把阅读区相关容器背景统一成纸色，消除空着时露出的白块。
- 手机上**顶栏导航不再挤成一团/被裁** → 顶栏换行、导航可横向滑动。
- 阅读区在手机上字号/边距更舒服。

还需要在真机上继续调（我在沙箱里跑不了服务、也没真机，没法验证这两块）：
- **特别卡**：主要卡在书房——你这有 6 万多本书，一排排全渲染出来手机扛不住。真正解决要做"虚拟滚动/分页加载"（只渲染屏幕内的书），这是**架构级改造**，我可以下一步专门做。
- **长篇没有翻页动画**：翻页动画库在超长篇（几百页以上）会自动退回"无动画的简单翻页"以防卡死，所以你点开的长篇看不到翻书效果。手机上让长篇也有动画需要换一套更轻的翻页实现，得对着真机调。
