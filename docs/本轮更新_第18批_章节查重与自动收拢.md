# 本轮更新（第 18 批）— 章节优先的精确查重 + 自动收拢散落的小说

按你说的"整个做完再给"，这两件都做好了，且都按**先预览、可撤销/不销毁**来设计。

## 1. 先补章节、确定章节后再精确查重 ✅
查重挑"保留哪一版"时，现在会**优先用章节数**（最全 = 章节多 + 字数多）：
- 如果你**先跑过「完结识别 / 查章节」**（它会把每本的章节数、番外数存进库），查重就直接拿这份数据来比——**更准、而且不再只对前 400 组生效，全部重复组都按章节挑最全的**。
- 没跑过完结识别的，查重会现场数一部分（仍有上限，避免太慢）。
- 防误数：番外 > 60、章节 > 3000 视为数错，按 0 处理（不会因为某本被误数出几百"番外"而留错版本）。
- 排序优先级：收藏 > 字数 > 章节 > 文件大小 > 番外。
- 扫描完会提示：如果库里章节数据覆盖不足 30%，会建议你"先跑完结识别再查重"。
- 去重一如既往**不删文件**，只移到隔离区，可在「版本管理」里找回。

操作：管理页 → 先点「完结识别 / 查章节」跑完 → 再「扫描重复」→ 看报告 → 「执行去重」。

## 2. 自动收拢散落的小说（微信/QQ/下载/桌面）✅
管理页新增卡片「**自动收拢散落的小说**」：
- 自动扫这些常见位置：**微信**文件接收（新版 xwechat_files / 旧版 WeChat Files）、**QQ/TIM** 接收（Tencent Files/FileRecv）、**下载**、**桌面**。
- 把里面的小说（txt/epub/压缩包）找出来入库，**自动去重**（和现有库比哈希，重复的跳过）。
- 两个按钮：
  - **预览（不动文件）**：只扫描、告诉你大概能新增多少本，**不碰任何文件**。
  - **执行入库（移动原文件）**：把找到的小说**移动**进书库（从微信/下载等原处移走）。点之前有二次确认。
- 说明：是"移动进书库"，不是销毁——原文件只是从散落的地方搬进了你的书库。
