# 本轮更新（第 24 批）— 手机端排版重做 + 书单拖拽 + 服务端分页

把剩下两项做了，并重做了手机端排版。

## 1. 手机端排版重做 ✅
桌面是大屏布局，手机上挤。这轮按窄屏重新排：
- **顶栏**：导航横向可滑、不再被裁；搜索框独占一行，"按作者/按题材/选择"落到下一行。
- **书详情弹窗**：接近整屏、可滚动；底部「阅读/移出/收藏/下载/编辑」自动换行，不再挤成一条细缝（"阅读"占整行更好按）。
- **手账本 / 书单卡片**：收紧留白、占满屏宽，标题字号适配。
- **阅读历程的标签**可横向滑动。
- **封面尺寸**随屏宽自适应（窄屏更小排得下），**Quill 球**缩小不挡内容。

## 2. 书单：可拖拽排版 ✅
在「分享书单」里，**标题区可以直接拖动**到卡片任意位置（手机触摸也行），导出的图片会按你拖的位置走；拖过之后出现「重置标题位置」。
> 说明：这是"自由拖拽"的第一步（先做了标题块）。要把每一本、每个元素都能任意拖，是完整画布编辑器，工程更大；目前标题拖动 + 背景图 + 密度 + 配色 + 显隐 + 筛选已能拼出很多样式。

## 3. 手机端服务端分页 ✅
`/api/shelf` 现在支持分页（`offset`/`limit`）。**手机窄屏**首次打开时：先取第一页（约 1200 本）**秒开**，剩下的在**后台**继续取、取完并入（不打断你正在看的位置）。这样 6 万本不再卡首屏。桌面端维持一次取全（搜索/筛选仍在本地，体验不变）。
> 完整版的"服务端搜索 + 无限滚动"是更大的改造；这版用"先一页秒开 + 后台补全"拿到手机端的主要收益，且不动现有稳定的搜索/筛选逻辑。
