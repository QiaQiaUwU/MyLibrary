# 本轮更新（第 29 批）— 找到"改了却看不到"的真正原因

## 真正的根因（重要）
你前面几批"没生效"，不是我没改，是**你打开的还是旧服务**：
1. 服务端启动时把前端文件（书房/阅读器的 HTML、JS）**一次性读进内存**，之后即使你覆盖了磁盘上的代码，它还吐内存里的旧版。
2. 用 pythonw 启动**没有窗口**，旧的服务进程可能一直在后台占着 8765 端口；你再点启动，新进程绑不上端口直接退出，浏览器打开的还是那个**旧服务** → 看到的当然是旧界面。

所以第 29/31 批里翻页、去掉三个点、Quill/声音可拖动这些代码其实都在，只是没送到你眼前。

## 这批的修复
- **前端实时读盘**：每次刷新页面，服务都从磁盘重新读最新的 HTML/JS。以后覆盖代码、刷新就能看到新版。
- **启动自动清掉旧实例**：启动时若发现 8765 端口被旧进程占着，先把它关掉再起（"最新启动的说了算"），保证你用的是最新代码。
- **图标换成 App 里那只羽毛笔**：桌面快捷方式的图标改用和右下角 Quill 完全相同的羽毛笔轮廓，不再是之前那个像叶子/咖啡豆的。
- **声音小窗排版加固**：标题栏「全部停止」改成「静音」更紧凑；小窗限制在窗口内、内容多了能滚动，不会再把字挤得看不见。

## 第一次更新请这样做（只此一次）
旧的后台服务还在跑、而且它**没有**自动清理功能，所以第一次要手动让它退出：
1. 用 v32 覆盖 D:\MyLibrary_v3。
2. 打开**任务管理器**，结束所有 `python.exe` / `pythonw.exe`（或者干脆重启电脑）。
3. 再双击启动 / 桌面快捷方式。
之后再更新就不用这么麻烦了——新版启动时会自动把旧的关掉。
