# 本轮更新（第 34 批）— 终于抓到"无黑窗点不开"的真凶

## 真凶
你那个弹窗的报错说得很清楚：
`AttributeError: 'NoneType' object has no attribute 'isatty'`
—— 用 **pythonw（无黑窗）** 启动时，`sys.stdout` 是 **None**；而 uvicorn 启动日志时会执行
`sys.stdout.isatty()`，于是直接崩溃，服务起不来 → 快捷方式"点不开"。
（用 启动.bat 有黑窗时 sys.stdout 正常，所以那种方式能跑——这也解释了之前的差别。）

## 修复
启动时如果发现 `sys.stdout / sys.stderr` 是 None（pythonw 情况），就给它们接上一个真实的流
（写到项目目录的 `mylib_log.txt`，顺便留个运行日志）。这样 `isatty()` 正常返回，uvicorn 不再崩。
已用模拟验证：接上文件后 `isatty()` 返回 False，uvicorn 日志初始化通过。

## 你现在该能用了
1. 用 v37 覆盖；先在任务管理器结束所有 python.exe / pythonw.exe。
2. 双击桌面那个 **MyLibrary**（pythonw、无黑窗）。约 1 秒后独立窗口自动弹出。
3. 万一还有别的问题，根目录的 `mylib_log.txt` 会有运行日志，启动失败也仍会弹对话框。

> 这次是真正的启动崩溃修复，不是绕路。先试这个无黑窗快捷方式。
