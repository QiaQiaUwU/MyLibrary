## 五、风险与建议

拆分本身不增加功能，纯粹是"把代码理顺"。风险点：
1. 89 个路由共享全局 `LIB/CONFIG/TASK_MANAGER`，拆到多文件要用统一的依赖注入或共享模块，拆错会路由失效
2. 内嵌 HTML 里 Python f-string 的 `{var}` 和前端 JS 的 `${var}` 混在一起，剥离时要小心转义
3. 每拆一块都要立刻跑一次端到端测试，确保 4 个页面 + 主要 API 不白屏

**建议节奏**：阶段 A 和 B 各自分 1 轮做完、每步都测；C、D 可以合并 1 轮。这样每轮交付都是能跑的完整包，不会卡在半成品。
