整理流水线 · 推荐顺序（由浅入深，一步步来）
第一次整理大库建议按这个顺序走；每步都「先预览、后执行」，作者类操作执行前会自动备份数据库。点步骤跳到对应卡片。
  1. 1 入库把散落的书都收进库（批量入库 / 自动收拢）
  2. 2 完结 + 分章节全库扫一遍，存完结状态和章节数（后面去重择优、对比都靠它）
  3. 3 找回作者·按书名佚名书从书名里把作者提出来
  4. 4 找回/合并作者·按正文扫正文里的微博@/lofter/by，把同一作者不同笔名归一
  5. 5 整理作者名把"淮上-补车番"这类杂名清成核心作者名
  6. 6 去重作者/章节都齐了，再精确去重、留最全的版本
  7. 7 打标签去重后再用 AI 打标签，省 token（同一本不重复花钱）

入库 · 收集散落

批量入库（扫描文件夹）

填一个文件夹的完整路径，把里面的小说（txt/epub，含子文件夹、压缩包）一次性扫描入库——自动解压、去重。也可以直接把文件拖到「书房」页面导入少量书。
想"放进某文件夹就自动入库"（监听微信/下载/投递口）：在命令行跑 python mylib/tools/watch_inbox.py 你的库路径 --all（需 pip install watchdog）。

自动收拢散落的小说 本地

自动扫描常见位置——微信 / QQ 的文件接收目录、下载、桌面——把里面的小说（txt/epub/压缩包）找出来入库（自动去重）。建议先「预览」看看能收多少，确认后再「执行入库」。
⚠️「执行入库」会把找到的小说移动进书库（从微信/下载等原处移走，不是复制；原文件不会被销毁，只是搬进了书库）。先点「预览」不会动任何文件。

版本管理（重复副本）

去重时被判为副本、移到隔离区的文件，按保留的主版本分组。可把某个副本「找回」（拷回 books/_restored 并重新入库）。

备份信息表（导出 / 导入）

把每本书的「书名 / 作者 / 题材标签 / 评分 / 状态 / 笔记」导出成 JSON 备份；重装或换库后导入回填（按 书名+作者 或 原文件名 匹配）。也支持导入 Moon+ Reader 的 .mrpro。

查重 · 去重 · 版本

去重 本地·免费

三层去重：内容哈希相同 / 同作者书名相似 / 空壳残章。先扫描看报告，确认后执行。重复文件移到隔离区，不删除。
想更准：先跑「完结识别 + 分章节」——它会统计每本的章节数，查重挑"保留最全版本"时优先用章节数（更准、且不限组数）。
放心：去重删旧版本前，会把你在那本上写的简评、评分、收藏、已读、最近阅读、标签、划线/书签/笔记/进度自动搬到保留的那本，不会丢。

分类标签 · 作者整理

AI 内容打标签 需 API·按量计费

AI 读取每本书开头内容，判断真实体裁（耽美/古代/修真/快穿等）。建议先去重再打标，避免给重复书重复分析。

作者别名合并 本地·免费

自动检测同一作者的不同名字（如 江洋大刀/狄醉山、xxx_1 后缀变体）并归一。先预览规则数，确认后执行。

从书名找回作者 本地·免费

很多书入库时丢了作者，被标成"佚名"，但书名里常带着作者名（如"书名_诗无茶""书名 作者诗无茶""书名by诗无茶"）。这个任务扫描佚名书的书名，把作者找回来归并。先预览，确认后执行。

整理作者名 / 解析书名 本地·免费

入库时常把书名里的描述当成作者，导致同一作者被拆成一堆"杂名"：如「淮上-补车番 / 淮上肉章 / 淮上近代架空」其实都是 淮上；「priest完结番外三篇 / priest完结+番外」都是 priest；「火风L3p / 火风L换攻」都是 火风。这个任务把这些清成核心作者名并合并到一起，作者分类、查重、找回都会更准。先预览，确认后执行（自动备份数据库）。

按正文找回 / 合并作者（笔名归一）本地·免费

同一作者用了不同笔名入库时，光看作者名看不出是同一人，但正文开头常带身份标记（微博@、lofter、by XX、作者：XX、首发平台）。这个任务扫每本书正文开头，把"标记相同但作者名不一致"的书归一到同一作者——比如「抗病毒口服液」和「送泥一条鱼」靠正文里的微博@认出是同一人并合并。先预览，确认后执行（自动备份数据库）。
注意：要逐本读正文开头，13 万本会跑几分钟。

找回丢失的简评（从备份）本地·免费

如果你写过的简评/总评后来不见了，这里会两头找：①扫数据库的历次备份（library.db.bak*）；②查同名书的其它副本——很多时候总评其实写在了这本书的另一个重复副本上（还没去重），这就能把它补给你点开看到的那本空的。先预览，不会覆盖你现在已有的简评。

同名书笔记互通（更全的版本沿用旧笔记）本地·免费

你中途找到更全的版本（书名带「精校/全本/番外」等也能认出是同一本）时，这里把你在旧版本上写的划线、笔记、书签、简评、评分、收藏、已读、进度复制到同名书里还空白的那些版本上。
非破坏：两个版本都保留，只往空的补、不动已有数据的；执行前自动备份。划线/进度按字符位置走——"更全版本"开头通常和旧版一致，所以基本能对上（位置对不上的那条不显示而已，不会乱）。先预览。

带走补全核查 本地·免费

对比母库和平板 mrpro，列出哪些作者在母库有很多书但平板没带全。

完结识别 + 分章节 本地·免费

扫描每本书的章节和结尾，识别完结/连载状态。支持中断续跑：扫到一半关掉服务也没关系，下次再点"继续扫描"会自动跳过扫过的，从断点接着扫。13万本可以分几次扫完。
