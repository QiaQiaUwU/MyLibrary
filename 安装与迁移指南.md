# 安装配置 + 能不能删旧 mylib —— 看这一份就够

## 先搞清楚两样东西（关键！）

你电脑上其实是**两个独立的东西**，别搞混：

1. **代码文件夹**（你说的 `F:\mylib`）—— 程序本身。这次重构后**整个被新版替代**。
2. **数据文件夹**（`F:\MyLibrary`）—— 你 13 万本书、`library.db` 数据库、封面、主题图。
   **这个绝对不能动，新代码还要读它。**

> 注意大小写：`mylib`（代码）和 `MyLibrary`（书库数据）是两个不同的文件夹。
> 新代码包解压出来也叫 `MyLibrary_v3`——这是代码，不是你的书库数据。建议把它放到别处，别和书库混。

---

## 一、配置完整需要哪些东西

新代码包（仅代码版）解压后已经包含了**除音频外的全部**：代码、前端、配置模板、文档。你只需要补两类东西：

### 1. 补音频（仅代码版才需要；完整版已含，可跳过）
把你之前的白噪音和翻页音放进去：
- 白噪音 mp3 → 放进 `MyLibrary_v3\static\_ambient\`（文件名随意，界面按文件名显示）
- 翻页音 → 放进 `MyLibrary_v3\static\_sfx\pageturn.mp3`（必须叫这个名）

（你之前发我的 12 个音频，完整版里已按中文名放好了：雨声/海浪/篝火/咖啡馆/林间溪流/夏雨/雷雨/静雨/森林鸟鸣/林间风/轻海浪 + pageturn）

### 2. 填配置 config.json
解压后根目录有 `config.json`，用记事本打开，改两处：
```json
{
    "server": {"host": "0.0.0.0", "port": 8765, "open_browser": true},
    "library": {"root": "F:\\MyLibrary"},        ← 改成你书库的真实路径（注意双反斜杠）
    "ai": {"provider": "deepseek",
           "api_key": "你的-API-key",            ← 填你的 AI key（不填则 AI 功能不可用，其他正常）
           "base_url": "https://api.deepseek.com/v1",
           "model": "deepseek-chat"}
}
```
> 路径里的反斜杠要写两个：`F:\\MyLibrary`。
> 不填 api_key 也能跑，只是 Quill 对话、AI 打标签、体裁分类用不了；去重、作者归一、阅读、管理这些本地功能不受影响。

## 二、启动

在 `MyLibrary_v3` 文件夹里：
```
pip install -r requirements.txt    （只需第一次）
python main.py                     （读 config.json 里的路径）
```
或双击 `MyLibrary.bat`。浏览器开 http://127.0.0.1:8765

---

## 三、关于 mylib_core.py 和 mylib_html.py（重要）

这两个文件是**数据层底层 + 旧版页面**，我打包时用的是**之前聊天里的版本**。如果你本地的 `mylib` 里这两个文件有更新（比如你后来自己改过），请用**你本地的版本覆盖**包里的：
- 你的 `mylib_core.py` → 覆盖 `MyLibrary_v3\mylib\core\mylib_core.py`
- 你的 `mylib_html.py` → 覆盖 `MyLibrary_v3\mylib\server\mylib_html.py`

如果你没单独改过它们，那包里的就能用，不用管。

## 四、能不能删旧的 mylib？—— 分三步，确认后再删

**不要急着删。按这个顺序来：**

### 第 1 步：先跑通新版
按上面启动，确认：
- 书房首页能打开、能看到你的书
- 点开一本书能阅读、能翻页
- 管理页能看到统计数字（总书目 13 万多）
- 阅读历程、收藏翻阅正常

只要这些都正常，说明新代码 + 你的 `F:\MyLibrary` 数据配合无误。

### 第 2 步：从旧 mylib 抢救可能漏掉的东西
打开你的旧 `F:\mylib`，看看有没有这些**不在新代码包里、但你可能想留**的：
- `*.mrpro` 文件（你的分类/标签数据导出，如 `2026-06-07-classified.mrpro`）→ 如果还用得到，复制出来留存
- 任何你自己写的脚本 / 笔记 / 聊天记录 txt
- `mylib_core.py` / `mylib_html.py`（如果你改过，见上面第三节）

把这些有用的挑出来另存。

### 第 3 步：确认后再删
当 1、2 步都做完、新版连续用几天没问题，就可以删旧的 `F:\mylib` 代码文件夹了。

> **再强调一次：删的是代码文件夹 `mylib`，绝对不要删数据文件夹 `MyLibrary`。**
> 删之前最稳妥的做法：先把旧 `mylib` 改名成 `mylib_old` 放一周，确认没事再彻底删。

---

## 五、一句话总结

- 数据（`F:\MyLibrary`）：永远别动。
- 旧代码（`F:\mylib`）：先抢救里面的 .mrpro 和你改过的文件，新版跑通几天没问题后可删（建议先改名留一周）。
- 新代码（`MyLibrary_v3`）：填好 config.json 的库路径和 api_key，补好音频，`python main.py` 启动。
